# AGENTS.md

## Zig std 開發準則

- 本 Zig 專案實作時，一律以本機 `std/` 資料夾作為首要參考來源。
- 需要確認 Zig 標準庫、lib 路徑或編譯環境時，優先使用 `zig env` 取得實際路徑與設定。
- 查閱 `std/` 時，應以 `zig env` 顯示的本機 Zig 安裝內容為準，不假設固定安裝路徑。
- 不得憑印象猜測 Zig std API。
- 若涉及 std API、型別、error set、allocator 行為、I/O、async、build system，必須先查本機 `std/` 原始碼或註解。
- 若找不到明確依據，應明講不確定，並先提出可驗證方式再實作。

## 實作原則

- 標準庫優先，避免不必要的第三方依賴。
- 先確認語意與生命週期，再寫程式，尤其是 allocator、檔案控制代碼、process、buffer。
- 對錯誤路徑與資源釋放要明確，不依賴隱含行為。
- 若 API 行為可能因 Zig 版本不同而有差異，應先比對本機版本對應的 `std/` 實作。

## 回覆與驗證

- 說明 Zig 行為時，優先引用本機 `std/` 可驗證的依據，而不是記憶中的寫法。
- 若需要試驗，先提出最小可重現範例或驗證步驟，再進行實作。
- 遇到不確定的 std 行為時，不硬做；先縮小問題，再確認依據。
